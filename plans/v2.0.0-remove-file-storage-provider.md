# Plan: Remove FileStorageProvider - v2.0.0

**Created**: 2026-01-31
**Target Version**: 2.0.0 (Major - Breaking Change)
**Status**: Draft

---

## Executive Summary

Remove the deprecated `FileStorageProvider` and all file-based storage fallbacks from the codebase. This is a breaking change that requires a major version bump to v2.0.0. Neo4j becomes the sole storage backend, aligning the codebase with its stated purpose as a Neo4j knowledge graph MCP server.

---

## Rationale

### Why Remove FileStorageProvider

1. **Feature Disparity**: FileStorageProvider cannot support core features:
   - No temporal versioning (entity/relation history)
   - No vector embeddings or semantic search
   - No graph traversal optimization
   - No concurrent access safety

2. **Already Deprecated**: Marked deprecated since early versions with warnings

3. **Misleading API**: Offering `type: 'file'` suggests parity with Neo4j, which doesn't exist

4. **Maintenance Burden**: ~434 lines of code + 5 test files requiring maintenance

5. **Coverage Drag**: 68% coverage on deprecated code hurts metrics

### Why v2.0.0

Per semantic versioning, removing a public API (the 'file' storage type) is a breaking change requiring a major version increment.

### Phase 3: Documentation Updates

#### Step 3.1: CHANGELOG.md

````markdown
## [2.0.0] - YYYY-MM-DD

### Breaking Changes

- **REMOVED**: `FileStorageProvider` class and all file-based storage
- **REMOVED**: `type: 'file'` option in `StorageProviderConfig`
- **REMOVED**: `memoryFilePath` option in `KnowledgeGraphManagerOptions`
- **REMOVED**: `MEMORY_FILE_PATH` environment variable support
- **REQUIRED**: Neo4j database is now required for all usage

### Migration Guide

If you were using file-based storage:

1. Install Neo4j (Docker recommended):
   ```bash
   docker run -d --name neo4j \
     -p 7474:7474 -p 7687:7687 \
     -e NEO4J_AUTH=neo4j/password \
     neo4j:5.26-community
   ```
````

2. Update your configuration:

   ```typescript
   // Before (v1.x)
   const manager = new KnowledgeGraphManager({
     memoryFilePath: './memory.json',
   });

   // After (v2.x)
   const provider = new Neo4jStorageProvider({
     config: {
       uri: 'bolt://localhost:7687',
       username: 'neo4j',
       password: 'password',
     },
   });
   const manager = new KnowledgeGraphManager({
     storageProvider: provider,
   });
   ```

3. Migrate existing data:
   - Export from JSON file
   - Use `createEntities` and `createRelations` to import

